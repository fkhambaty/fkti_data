⚡ Performance Tuning

-- Check query execution plan
EXPLAIN SELECT * FROM orders WHERE order_date > '2024-01-01';

-- Find slow queries
SELECT query, elapsed, substring
FROM svl_qlog
ORDER BY elapsed DESC
LIMIT 10;

-- Check table statistics
SELECT * FROM svv_table_info WHERE "table" = 'orders';

-- Vacuum and analyze (maintenance)
VACUUM orders;
ANALYZE orders;

⚠️ Top Performance Killers

  • Wrong DISTKEY → data skew → one node does all the work
  • Missing SORTKEY → full table scans on filtered queries
  • Not running VACUUM → dead rows waste space
  • Too many concurrent queries → use WLM queues
