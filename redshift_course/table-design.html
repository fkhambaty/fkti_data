🔑 The Three Pillars of Table Design

1. Distribution Key (DISTKEY)

Determines which node stores each row. Choose the column you JOIN on most often.

CREATE TABLE orders (
    order_id BIGINT,
    customer_id BIGINT DISTKEY,
    order_date DATE SORTKEY,
    total DECIMAL(10,2)
);

2. Sort Key (SORTKEY)

Determines the physical order of rows on disk. Choose the column you filter (WHERE) on most often.

3. Compression Encoding

Redshift auto-selects the best compression. Use ANALYZE COMPRESSION to verify.

🧒 ELI5: The Library System

DISTKEY = Which building (node) to store the book in. SORTKEY = How to arrange books on the shelf (by date? by author?). Compression = How tightly to pack the books. Get all three right and finding any book is instant!
