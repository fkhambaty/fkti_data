⚙️ Creating Your First Cluster

🧒 ELI5: Renting Computers

Creating a Redshift cluster is like renting a team of computers from Amazon. You choose how many computers (nodes) you want and how powerful they should be. Amazon sets them up, connects them together, and gives you a URL to connect.

Go to AWS Console

Navigate to Amazon Redshift in the AWS Management Console. Click "Create cluster."

Choose Node Type

dc2.large ($0.25/hr) for testing. ra3.xlplus for production — separates compute from storage.

Configure Network

Choose your VPC, subnet, and security group. Enable "Publicly accessible" for learning (disable in production!).

Set Credentials

Create an admin user and password. You'll use these to connect from SQL clients.

Connect!

Use the JDBC/ODBC endpoint to connect from DBeaver, DataGrip, or psql.

-- Connect using psql (Redshift is PostgreSQL-compatible!)
psql -h my-cluster.abc123.us-east-1.redshift.amazonaws.com \
     -p 5439 -U admin -d dev

⚠️ Cost Warning

Redshift clusters cost money even when idle! Always pause or delete your cluster when not using it. For learning, use Redshift Serverless — you only pay for what you query.
