âœ¨ Customized Line Chart (The Pro Way!)

import numpy as np
import matplotlib.pyplot as plt

# Create two datasets
x = np.arange(0, 10, 1)
y = 2*x + 3
z = 3*x + 2

# Create a larger figure
fig = plt.figure(figsize=(9, 8))

# Plot first line with customizations
plt.plot(x, y, 
         linestyle='-.', 
         marker="^",      # Triangle markers
         markersize=14,   # Big markers
         color="m",       # Magenta color
         alpha=0.9)       # Slight transparency

# Plot second line
plt.plot(x, z, 
         linestyle='-', 
         marker="8",      # Octagon markers
         markersize=10, 
         color="r",       # Red color
         alpha=0.3)       # More transparent

# Add labels to each point
for i in range(len(x)):
    plt.annotate(f"({x[i]},{y[i]})",  # Text to display
                 (x[i], y[i]),         # Point to annotate
                 textcoords="offset points",
                 xytext=(0, 10),       # Offset above marker
                 ha="center")

# Add labels and title
plt.xlabel("price")
plt.ylabel("area")
plt.title("Line Chart")
plt.grid(True)
plt.legend(["xy", "xz"], loc="best")

# Save the figure
plt.savefig("line_chart.jpg")
plt.show()

ğŸ“Š Bar Charts: Comparing Categories

ğŸ† Real-World Example: Election Results

Bar charts are like podiums at the Olympics! Each bar represents a category (candidate, product, team), and the height shows the value (votes, sales, score). Taller bar = better performance!

Bar charts answer: "Which category is bigger/smaller?"

ğŸ“ Bar Chart Anatomy

  • X-axis: Categories (education, loan status, etc.)
  • Y-axis: Values (counts, amounts, percentages)
  • Bars: Height represents magnitude
  • Colors: Can show sub-categories

ğŸ“± Subplots: Multiple Charts in One Figure

ğŸ“º TV Analogy: Split Screen

Subplots are like a split-screen on your TV! Instead of switching between channels, you see multiple shows at once. Data scientists use this to compare different aspects of data side-by-side.

ğŸ”¢ Understanding subplot(rows, cols, position)

  • subplot(2, 2, 1) = 2 rows, 2 cols, position 1 (top-left)
  • subplot(2, 2, 2) = 2 rows, 2 cols, position 2 (top-right)
  • subplot(2, 2, 3) = 2 rows, 2 cols, position 3 (bottom-left)
  • subplot(2, 2, 4) = 2 rows, 2 cols, position 4 (bottom-right)

ğŸ“Š What's a Histogram with KDE?

  • Histogram: Bars showing how many values fall in each range
  • KDE (Kernel Density Estimate): The smooth curve overlaid showing the distribution shape
  • Together they answer: "What's the typical value? Are there outliers?"

âš ï¸ Seaborn v0.14+ FutureWarning Fix

Starting from Seaborn v0.14, passing palette without hue is deprecated. Here's how to fix it:

âŒ OLD (Deprecated - will show warning): 
# This will show FutureWarning in Seaborn v0.14+
sns.violinplot(x='loan_status', y='cibil_score', data=df, palette='muted')
sns.boxplot(x='category', y='value', data=df, palette='Set2')
sns.barplot(x='group', y='score', data=df, palette='viridis')
âœ… NEW (Correct way): 
# Assign the x variable to hue and set legend=False
sns.violinplot(x='loan_status', y='cibil_score', data=df, hue='loan_status', palette='muted', legend=False)
sns.boxplot(x='category', y='value', data=df, hue='category', palette='Set2', legend=False)
sns.barplot(x='group', y='score', data=df, hue='group', palette='viridis', legend=False)
ğŸ’¡ Why this change?

Seaborn wants to be explicit about which variable controls the colors. By assigning hue to the same variable as x, you're explicitly saying "color by this variable". The legend=False prevents a redundant legend since x-axis already shows the categories.

ğŸš« Common Mistakes (Data Viz)

  • Wrong chart for the question â€” Use line for trends over time, bar for comparing categories; pie only for few parts of a whole.
  • Missing labels or title â€” Always add xlabel, ylabel, and a title so the reader knows what they're looking at.
  • Plotting raw data without aggregation â€” For many points, aggregate (e.g. mean by group) or use a plot that handles density (e.g. hexbin, kde).
