Installation & SSMS

Download SQL Server (Developer or Express) and SQL Server Management Studio. Install step-by-step and connect for the first time!

Step 1: Download SQL Server

You need the database engine (where data is stored) and optionally other features. For this course, Database Engine Services is the minimum.

Download

1
Go to Microsoft's download page

Open Microsoft SQL Server Downloads. Choose Developer (recommended) or Express.

2
Run the installer

Run the downloaded file (e.g. SQL2019-SSEI-Dev or similar). Choose Custom install so you can select only what you need.

Step 2: Install SQL Server (Database Engine)

During setup:

  • On Feature Selection, check Database Engine Services. That's the minimum to use SQL Server.
  • On Instance Configuration, keep Default instance unless you need multiple instances.
  • On Database Engine Configuration, choose Mixed Mode and set a strong password for the sa account. Click Add Current User as SQL Server administrator.

Important

Use Mixed Mode so you can log in with both Windows and the sa (SQL admin) account. Remember the sa password—you'll need it to connect with SQL Server Authentication.

Step 3: Download & Install SSMS

SQL Server Management Studio (SSMS) is the client tool you use to connect to SQL Server and run queries. Install it after SQL Server.

Get SSMS

1
Download SSMS

Search for "Download SQL Server Management Studio" or go to the link provided at the end of the SQL Server installer. Download the latest SSMS and run the installer.

2
Install

Click Install and wait for setup to finish. No need to change default options for this course.

Step 4: Connect to SQL Server

Open SQL Server Management Studio. In the connection dialog:

  • Server type: Database Engine
  • Server name: Your computer name, or localhost, or . (dot) for default instance.
  • Authentication: Windows Authentication (uses your Windows login) or SQL Server Authentication (login: sa, password: the one you set).

Click Connect. If you see Object Explorer on the left, you're in!

Key Takeaway

SQL Server = the engine (data lives here). SSMS = the tool you use to talk to the engine. You must install both and then connect from SSMS to your server.

Mac Users

SQL Server runs on Windows and Linux, not natively on macOS. Options: (1) Use a Windows virtual machine (e.g. Parallels, VirtualBox) and install SQL Server inside it. (2) Use SQL Server in Docker on your Mac. (3) Use a cloud or remote Windows machine. Many courses and companies use Windows for SQL Server administration.

What You're Installing

SQL Server (Engine) — runs as a Windows/Linux service. Stores databases and runs your T-SQL.

SSMS (Client) — the app you open. You type queries here and click Execute; the engine runs them and returns results.

No Code Yet!

This module is about installing tools. In Module 3 you'll open a New Query window in SSMS and run your first T-SQL (e.g. SELECT DB_NAME();). For now, just get SQL Server and SSMS installed and connect once.

Practice

Checklist

□ Download SQL Server (Developer or Express) from Microsoft.

□ Run setup and select Database Engine Services; use Mixed Mode and set sa password.

□ Download and install SQL Server Management Studio (SSMS).

□ Open SSMS and connect to your server (Server name: . or localhost; Authentication: Windows or SQL Server with sa).

□ Confirm you see Object Explorer on the left. You're ready for Module 3!
