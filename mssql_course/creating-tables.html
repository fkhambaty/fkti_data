Creating Tables & Data Types

Define tables with CREATE TABLE: column names, data types, PRIMARY KEY, and NOT NULL.

CREATE TABLE

Use CREATE TABLE to define a new table: give it a name and list each column with a name and data type.

Column names

Column names must be unique within the table. Use descriptive names (e.g. CustomerId, OrderDate). Avoid spaces; use PascalCase or snake_case.

Data types

TypeUse for
INTWhole numbers (e.g. IDs, counts)
VARCHAR(n)Variable-length text, max n characters
DATEDate only (no time)
DECIMAL(p,s)Exact numbers (p = total digits, s = decimal places)

PRIMARY KEY & NOT NULL

PRIMARY KEY identifies each row uniquely. Only one primary key per table; it implies NOT NULL and unique values. NOT NULL means the column cannot store NULL—every row must have a value.

Key Takeaway

Always define a primary key for your main tables. Use NOT NULL on columns that must always have a value.

Table structure

A table is like a spreadsheet: columns (name + type) and rows. PRIMARY KEY marks the column(s) that uniquely identify each row.

CREATE TABLE example

-- Table with 3 columns: Id (PK), Name, BirthDate
CREATE TABLE Customers (
    Id        INT PRIMARY KEY,
    Name      VARCHAR(100) NOT NULL,
    BirthDate DATE NULL
);

Practice

Create a table Products with columns: ProductId (INT, PRIMARY KEY), ProductName (VARCHAR(200), NOT NULL), and Price (DECIMAL(10,2)).
