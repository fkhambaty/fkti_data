Meet SQL Server

What is a database? What is Microsoft SQL Server? Why do companies use it? Let's break it down from zero!

What is a Database?

A database is an organized place where we store data so we can add, change, and look it up later. Instead of millions of text files or spreadsheets, we use a database management system (DBMS) to keep everything in tables and views.

Explain Like I'm 5

Imagine a huge filing cabinet. Each drawer has folders (tables), and each folder has rows of information (like one row per customer or one row per order). When you want to find something, you don't flip every page—you ask the cabinet in a special language: "Give me all customers from New York." That's what a database does!

Organizations store data in tables (rows and columns) and often use views (saved queries that look like tables). Microsoft SQL Server is one of the most popular DBMSs in the world.

What is Microsoft SQL Server?

Microsoft SQL Server is a database management system by Microsoft. Companies use it to store data in tables and views, manage it with the SQL language, and run enterprise applications with high availability and security.

The Kitchen Analogy

SQL Server = The kitchen (the engine that stores and serves data).
Tables = Shelves and fridges (where ingredients/data live).
SQL (T-SQL) = The recipe language you use to ask for food: "Give me all tomatoes" (SELECT) or "Add a new tomato" (INSERT).
SSMS = The control panel where the chef (you) runs those recipes and sees results.

SQL Server supports many of the ANSI/ISO SQL standards, so what you learn here is portable. It runs on Windows and Linux, and scales from a single laptop to huge clusters and the cloud (Azure).

SQL Server Editions

Microsoft offers different editions. For learning and development, you only need to know two:

Editions You'll Care About

1
Developer Edition

Free, full-featured like Enterprise, but licensed only for development and test—not production. Perfect for learning and building projects.

2
Express Edition

Free, smaller version with limits on database size and features. Great for lightweight learning and small apps.

Other editions (Standard, Enterprise) are for production and have licensing costs. For this course, Developer or Express is enough.

The Three Languages of SQL

SQL is used to manage databases through three types of commands:

TypeFull NameWhat It DoesExample Commands
DDLData Definition LanguageCreates and changes the structure (tables, columns, indexes)CREATE, ALTER, DROP
DMLData Manipulation LanguageAdds, changes, and reads dataSELECT, INSERT, UPDATE, DELETE
TCLTransaction Control LanguageControls transactions (save or undo a set of changes)BEGIN, COMMIT, ROLLBACK

Key Takeaway

DDL = shape of the database. DML = the data inside. TCL = making sure multiple changes succeed or fail together. You'll use DML (especially SELECT) every day; DDL when you design or change tables; TCL when you need safe multi-step updates.

When to Use SQL Server?

Companies use SQL Server for:

  • Enterprise apps (banking, healthcare, retail) that need reliable, secure data
  • Reporting and business intelligence (dashboards, analytics)
  • Web and mobile backends that store users, orders, and content
  • High availability and disaster recovery (Always On, backups)

Real-World Use

SQL Server powers core systems at banks, hospitals, retailers, and tech companies. Stack Overflow, for example, uses SQL Server to serve millions of developers with sub-second search and high uptime. Learning SQL Server opens doors to jobs in data, backend development, and database administration.

How Data Flows

From you to the data and back:

→
T-SQL Query
→
SQL Server

Tables / Results

You type a command → SQL Server runs it → You see rows (or a message)

Data lives in tables; SQL Server reads/writes them and returns results to you.

DDL vs DML vs TCL

DDL — Structure

CREATE TABLE, ALTER TABLE, DROP TABLE. Changes the "shape" of the database (columns, keys), not the data inside.

DML — Data

SELECT, INSERT, UPDATE, DELETE. Reads and changes the actual rows in tables. This is what you'll use most.

TCL — Transactions

BEGIN, COMMIT, ROLLBACK. Groups several changes into one unit: either all succeed or all are undone.

Quick Syntax Preview

You don't run these in Module 1—they're just a preview of what DDL, DML, and TCL look like.

-- DDL: create a table (structure)
CREATE TABLE Customers (
    CustomerID   INT PRIMARY KEY,
    CustomerName VARCHAR(100),
    City         VARCHAR(50)
);

-- DML: add and read data
INSERT INTO Customers VALUES (1, 'Alice', 'New York');
SELECT * FROM Customers;

-- TCL: wrap changes in a transaction
BEGIN TRANSACTION;
UPDATE Customers SET City = 'Boston' WHERE CustomerID = 1;
COMMIT;

In later modules you'll create real tables, run SELECT queries, and use transactions. For now, just notice: DDL defines structure, DML touches data, TCL controls commits.
