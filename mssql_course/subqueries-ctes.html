Subqueries & CTEs

Use subqueries in WHERE and FROM, and name result sets with WITH ... AS (CTEs).

Subqueries

A subquery is a SELECT inside another query. It can appear in WHERE (e.g. WHERE Id IN (SELECT ...)) or in FROM (e.g. FROM (SELECT ...) AS sub).

Subquery in WHERE

Use it to compare against a set or single value: WHERE CustomerId IN (SELECT Id FROM Customers WHERE Country = 'UK') or WHERE Amount > (SELECT AVG(Amount) FROM Orders).

Subquery in FROM

Use a subquery as a derived table: SELECT * FROM (SELECT CustomerId, SUM(Amount) AS Total FROM Orders GROUP BY CustomerId) AS t. You must give it an alias (e.g. t).

CTE — WITH ... AS

A Common Table Expression (CTE) names a result set with WITH name AS (SELECT ...), then you query that name. Makes complex queries easier to read and reuse. Example: WITH TopCustomers AS (SELECT ...) SELECT * FROM TopCustomers.

Key Takeaway

Subqueries nest one query inside another. CTEs define a named result set first, then reference it in the main query.

Structure

Subquery: outer query wraps inner query (in WHERE or FROM). CTE: WITH defines a "virtual table" by name, then the main SELECT uses it.

Subquery and CTE examples

-- Subquery in WHERE: orders above average amount
SELECT * FROM Orders
WHERE Amount > (SELECT AVG(Amount) FROM Orders);

-- CTE: name the result set, then use it
WITH OrderTotals AS (
    SELECT CustomerId, SUM(Amount) AS Total
    FROM Orders GROUP BY CustomerId
)
SELECT c.Name, ot.Total
FROM OrderTotals ot
INNER JOIN Customers c ON c.Id = ot.CustomerId
ORDER BY ot.Total DESC;

Practice

A new physical table

Correct! WITH ... AS defines a Common Table Expression (CTE).

WITH ... AS defines a CTE: a named result set that the following SELECT can use like a table.
